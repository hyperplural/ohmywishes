# Ohmywishes API v3

Observed `v3` endpoints cover context discovery, public profiles, social graph, settings, auth, selections, wish lists, and reservations.

## Client Context

- `GET /api/v3/client`

## Content Regions

- `GET /api/v3/catalogs/content-regions`
- `GET /api/v3/catalogs/content-regions/{regionCode}`

## Public Profiles and Social Graph

- `GET /api/v3/users/{username}`
- `GET /api/v3/users/{userId}/followings`
- `GET /api/v3/users/{userId}/followers`
- `POST /api/v3/users/{userId}/following`
- `DELETE /api/v3/users/{userId}/following`
- `GET /api/v3/own-user/followers/requests`

## Notifications and Settings

- `GET /api/v3/own-user/notifications/settings`
- `PUT /api/v3/own-user/settings`

Notes:
- `PUT /api/v3/own-user/settings` controls privacy, subscription approval, and notification toggles.

## Email Flows

- `POST /api/v3/own-user/email/confirmation-code`
- `PUT /api/v3/own-user/email`
- `POST /api/v3/auth/email/confirmation-code`
- `POST /api/v3/auth/email/login`
- `POST /api/v3/auth/phone-number/confirmation-code`
- `POST /api/v3/auth/phone-number/login`

Notes:
- `own-user/email/*` is for changing the current account email.
- `auth/email/*` is the passwordless login flow.
- `auth/phone-number/*` is the phone-number code login flow.

## Selections

- `GET /api/v3/selections`
- `GET /api/v3/selections/{selectionId}/ideas?limit=20&offset=0`

## Wish Lists

- `GET /api/v3/own-user/wish-lists`
- `PUT /api/v3/own-user/wish-lists/{wishListId}`
- `PUT /api/v3/own-user/wish-lists/{wishListId}/position`

## Wish Actions

- `GET /api/v3/wishes/{wishId}`
- `POST /api/v3/wishes/{wishId}/reservation`
- `DELETE /api/v3/wishes/{wishId}/reservation`

## Notes

- `POST /api/v3/wishes/{wishId}/reservation` and `DELETE /api/v3/wishes/{wishId}/reservation` both return the updated wish payload.
