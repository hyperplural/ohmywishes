# Ohmywishes API v2

Observed `v2` endpoints focus on the authenticated user, owned wishes, wish aggregates, favorites, and reference data.

## Authenticated User

- `GET /api/v2/users/self`
- `PUT /api/v2/users/self`
- `POST /api/v2/users/self/avatars`

Notes:
- `POST /api/v2/users/self/avatars` uses `multipart/form-data` with `picture`.
- `PUT /api/v2/users/self` accepts profile fields such as `username`, `firstName`, `lastName`, `bio`, `sex`, and `birthday`.

## Own Wishes

- `POST /api/v2/users/self/wishes`
- `PUT /api/v2/users/self/wishes/{wishId}`
- `DELETE /api/v2/users/self/wishes/{wishId}`
- `POST /api/v2/users/self/wishes/{wishId}/picture`
- `POST /api/v2/wishes/{wishId}/copy`

Notes:
- `POST /api/v2/users/self/wishes` accepts `private` and `is_multi_reservation_available`.
- `POST /api/v2/users/self/wishes/{wishId}/picture` uses `multipart/form-data` with `picture`.
- `POST /api/v2/wishes/{wishId}/copy` returns a denormalized wish-list wish payload.

## Wish Aggregates

- `GET /api/v2/users/self/wish-lists/all/wishes?size=20&page=1`
- `GET /api/v2/users/self/wish-lists/private/wishes?size=20&page=1`
- `GET /api/v2/users/self/wish-lists/reserved/wishes?size=20&page=1`
- `GET /api/v2/users/self/wish-lists/fulfilled/wishes?size=20&page=1`

Notes:
- `private` returns hidden wishes.
- `reserved` returns wishes reserved by the current user.
- `fulfilled` returns fulfilled wishes.

## Favorites

- `POST /api/v2/users/{userId}/favorite`
- `DELETE /api/v2/users/{userId}/favorite`

## Reference Data

- `GET /api/v2/currencies`
