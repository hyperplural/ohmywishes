# Ohmywishes SDK Documentation

This SDK is based on the publicly observed Ohmywishes HTTP API.
It is unofficial and not affiliated with Ohmywishes.

## Read This First

- `docs/v2.md` covers authenticated user, wishes, wish lists, favorites, and reference data.
- `docs/v3.md` covers the newer client, public profiles, selections, reservations, settings, and auth flows.
- `docs/public.md` covers the public status endpoint.
- `docs/api-spec.md` contains appendix notes, normalization rules, and DTO observations.

## Structure

1. Read `docs/v2.md` for owned wish and profile flows.
2. Read `docs/v3.md` for the newer API surface.
3. Read `docs/public.md` for the service status payload.
4. Use `docs/api-spec.md` as the working appendix for field-level notes.
